使用hugo+gitpages搭建静态网站

January 13, 2019

参考文档：hugo getting-started

第一步：

安装hugo，这里以mac为例

  1. 以homebrew安装:
   brew install hugo
  1. 查看版本：
   hugo version

第二步：

创建一个静态站点：

hugo new site JohnnyTing.github.io

第三步：

  1. 下载主题，首先进入站点根目录：
   cd  JohnnyTing.github.io
  1. 下载主题，这里以ananke为例：
   git init  
   
   git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke
  1. 添加主题到config.toml配置文件：
   echo 'theme = "ananke"' >> config.toml

第四步：

添加内容（用markdown格式）：

hugo new posts/my-first-post.md

​ 生成的文件在content/posts/目录下，my-first-post.md文件去掉草稿，删除 draft：true 这行，因为创建内容默认是草稿模式，后面会介绍修改内容模板。

第五步：

​ 本地启动hugo服务，访问 http://localhost:1313/ 执行命令：

hugo server

第六步：

​ 生成静态网页到public文件夹，执行命令：

hugo

第七步：

  1. 在GitHub创建自己的仓库，username.github.io，这里我创建JohnnyTing.github.io仓库为例，username为你github账号的名字。

  1. 然后将自己的public本地网页文件关联到github仓库:
   cd public
   
   git init
   
   git add .
   
   git commit -m "first commit”
   
   git remote add origin https://github.com/JohnnyTing/JohnnyTing.github.io.git
   
   git push -u origin master
  1. 然后访问自己的静态网站：https://johnnyting.github.io/

第八步：

  1. 这里每次创建markdown文件后，都需要一系列的git操作 ，这里可以创建一个脚本文件deploy.sh来简化操作。
   touch deploy.sh
   chmod +x deploy.sh
  1. 给deploy.sh文件添加内容：
   #!/bin/bash
   
   echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"
   
   # Build the project.
   hugo # if using a theme, replace with `hugo -t <YOURTHEME>`
   
   # Go To Public folder
   cd public
   # Add changes to git.
   git add .
   
   # Commit changes.
   msg="rebuilding site `date`"
   if [ $# -eq 1 ]
     then msg="$1"
   fi
   git commit -m "$msg"
   
   # Push source and build repos.
   git push origin master
   
   # Come Back

  1. 新建文章：hugo new /post/first-post.md

  2. 执行脚本文件，然后推送到github仓库： ./deploy.sh

第九步：

  1. 添加Discus评论系统

    • 注册Discus（需要科学上网）> disqus官网

    • 首页点击 GET START ， 选择第二个 I want to install Disqus on my site

    • 在Website Name中输入一个名称，点击创建

    • Continue 正常配置

    • 配置页面有一个 Shortname Your website shortname is JohnnyTing.

    • 将你的shotname配置到config.toml

     DisqusShortname = “JohnnyTing”

    • 部署到GitHub上即可加载评论系统了

    • 需要注意的是，访问你博客的人需要科学上网才能加载评论。

  2. 在markdown文件中添加评论和分享功能

   comments: true     
   share: true
  1. 这里以我的config.toml示例:
